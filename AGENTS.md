# AGENTS.md

## 全局开发项目约束

以后在开发项目时，需要在合适的阶段持续配置和维护三个核心文件：

- `AGENTS.md`：项目的指令文件。如果没有需要 `/init` 产生。
- `README.md`：项目的介绍文件，也是项目进度文件。按照主流 README 风格写清楚项目定位、安装/运行方式、目录结构、当前状态、已完成进展和重要说明。
- `TODO.md`：项目待办和开发记录文件。记录还需要做的事情、阶段性开发记录、下一步行动、阻塞点和接续开发线索。

后续进入开发项目、新建项目、推进项目、接续项目、整理项目状态等场景时，应主动检查、创建或更新这三个文件，除非用户明确要求不要这样做。

开发新项目时，不能上来直接实现。应先使用 Superpowers 插件的头脑风暴能力梳理目标、约束、方案和风险，再进入项目创建、文件搭建或代码实现阶段。

## 本项目补充约束

- 本项目用于撰写一本以同级 `../warpx` 为核心入口的 PIC 程序讲解书。
- 默认只读使用 `../warpx` 源码、文档和示例，不修改 WarpX 原仓库内容。
- DeepWiki、Zread 等 AI 解读站点只作为索引和提纲线索；最终论断必须回到 WarpX 官方文档、本地源码、测试示例和可检索文献。
- 涉及物理、算法和代码实现的章节，必须记录对应源码路径、官方文档路径和参考文献线索。
- 阅读、理解和讲解论文时，按 `research-paper-explainer` skill 的形式执行：PDF 默认先用 MinerU 转 Markdown，中文笔记按论文顺序逐段总结，图片插在首次引用处，重要公式使用 `$$ ... $$` 并详细推导。
- 论文转换产物必须保存在论文所在目录或论文专属子目录内，包括原 PDF、MinerU Markdown、`images/` 和中文讲解笔记；不要把论文笔记集中散放到全局目录。
