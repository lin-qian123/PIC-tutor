# PIC-tutor 章节模板

每个正式章节都应尽量保留以下结构；短节可以合并，但不能省略证据链。

## 0. 章节状态

- 写作日期：

## 1. 本章目标

- 读者读完后应该能解释什么物理问题。
- 读者应该能在 WarpX 中找到哪些源码入口。
- 读者应该能运行或理解哪个示例。

## 2. 物理模型

连续方程、变量定义、适用近似、尺度条件、守恒律。

## 3. 离散算法

网格、时间层、形函数、插值、沉积、场求解器、稳定性限制和误差来源。

## 4. WarpX 实现

- 源码路径：
- 关键类 / 函数：
- 参数入口：
- 调用链：
- 必要行号：

源码讲解必须包含真实源码原文。推荐固定格式：

1. 先写“本段源码位置”，给出路径、函数名、行号范围、WarpX commit。
2. 放上对应源码块，使用 fenced code block，语言标为 `cpp`、`python`、`cmake` 或 `text`。
3. 源码块后逐行或逐块解释：每一组解释都要说明物理/算法含义、输入状态、输出状态、时间层、边界/guard cell 状态。
4. 只允许省略纯 boilerplate；省略时要明确说明“此处省略 include / profiling / 平台宏，不影响控制流”。

代码讲解必须先写伪代码，再放真实源码块，再解释真实源码块。不要离开物理和算法上下文单独讲 C++ 语法。
